Exercice Enum : Test arguments

Code source de l'énoncé

Dans TestArgument.main() (voir énoncé), on vérifie que l'utilisateur appelle cette classe avec un seul paramètre pouvant prendre les valeurs "ROOT", "ADMIN" ou "USER".

A l'utilisation, cela donne : 
java TestArguments 
Utilisation : java TestArguments <rôle>
<rôle> peut prendre les valeurs : ROOT, ADMIN, USER
il manque le paramètre rôle.

java TestArguments toto
Utilisation : java TestArguments <rôle>
<rôle> peut prendre les valeurs : ROOT, ADMIN, USER
Valeur incorrecte : toto

java TestArguments ROOT
OK, role = ROOT
Ecrire le code de vérification en utilisant l'API enum, sans coder en dur les chaînes "ROOT", "ADMIN" et "USER".

Vérifiez que si vous rajoutez un rôle (par exemple "VISITOR"), la fonction main() intègre ce nouveau rôle sans aucune modification.
